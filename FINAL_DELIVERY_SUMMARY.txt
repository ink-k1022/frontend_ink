🚀 立即開始使用
═══════════════════════════════════════════════════════════════════════════════

【方式 1】快速體驗（5分鐘）
    
    1. 取得 Google Maps API Key
       → https://console.cloud.google.com/google/maps-apis
    
    2. 編輯 config.js
       GOOGLE_MAPS_API_KEY: 'YOUR_KEY_HERE'
       ENABLE_MOCK_DATA: true
    
    3. 開啟 index.html
       python -m http.server 8000

【方式 2】完整設置（串接後端）
    
    1. 取得 Google Maps API Key
    
    2. 編輯 config.js
       GOOGLE_MAPS_API_KEY: 'YOUR_KEY_HERE'
       API_BASE_URL: 'http://localhost:3000/api'
       ENABLE_MOCK_DATA: false
    
    3. 啟動後端
       cd backend-examples/nodejs-express
       npm install
       npm start
    
    4. 啟動前端
       python -m http.server 8000

🔑 關鍵配置說明
═══════════════════════════════════════════════════════════════════════════════

【config.js】- 唯一需要修改的配置文件

    const CONFIG = {
        // 1. Google Maps API Key（必填）
        GOOGLE_MAPS: {
            API_KEY: 'YOUR_GOOGLE_MAPS_API_KEY_HERE'  // ← 改這裡
        },
        
        // 2. 後端 API 地址（必填）
        API: {
            BASE_URL: 'http://localhost:3000/api'     // ← 改這裡
        },
        
        // 3. 開發模式（測試時用）
        DEV: {
            ENABLE_MOCK_DATA: false  // true=不呼叫後端，使用模擬資料
        }
    };

🌐 後端 API 規格摘要
═══════════════════════════════════════════════════════════════════════════════

【必須實作的 3 個端點】

1. GET /api/venues
   參數: lat, lng, radius, category, minRating, openNow
   用途: 取得周邊店家列表

2. POST /api/venues/search
   Body: { query, lat, lng, radius, filters }
   用途: 搜尋店家

3. GET /api/venues/:id
   用途: 取得單一店家詳情

【標準回應格式】

    {
        "success": true,
        "data": [...],
        "message": "成功",
        "timestamp": "2026-02-07T14:30:00Z"
    }

【詳細規格】請參考: BACKEND_API_SPEC.md

🐛 常見問題快速解決
═══════════════════════════════════════════════════════════════════════════════

問題 1: Google Maps 不顯示
解決: 檢查 config.js 中的 API Key，確認已啟用 Maps JavaScript API

問題 2: CORS 錯誤
解決: 後端添加 CORS 設定，允許前端域名

問題 3: API 請求失敗
解決: 確認後端運行，檢查 config.js 的 BASE_URL

問題 4: 找不到店家
解決: 增加搜尋半徑，降低最低評分，或啟用模擬資料測試
